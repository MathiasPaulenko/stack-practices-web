# Thin Content Fix Checklist

## Context

StackPractices requires all content files to exceed **300 lines** to avoid thin content penalties. This checklist tracks files still below the threshold.

### Rules

- Bilingual parity: EN and ES files must have matching content and line counts
- Target 300-380 lines after expansion
- Add: more code examples, common mistakes, FAQ entries, deeper explanations
- Avoid AI-isms ("comprehensive guide", "leverage", "seamless", "delve into", etc.)
- Run `node scripts/thin-content-check.cjs` after each batch to verify
